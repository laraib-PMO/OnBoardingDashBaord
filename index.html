Signal Sources
All project signals from existing channels — no migration needed
Email BIM Voice Photos Files

Classify & Link
Each signal becomes a decision, obligation, deviation, or evidence — auto-linked to its project
03 · Process
AI Orchestration
6 specialized AI agents over a triple-store extract meaning and relationships from every signal

7 human-in-the-loop gates. No external or contractual action executes without a human approving

ProductPhase 0 active

Attimo is not a task manager and not a chatbot. It is a governed, evidence-native coordination layer. Every output cites its source. Every action is approval-gated. Every decision is logged immutably.

8 core user stories
US1
Recover context
Find the right info from vague or partial memory
US2
Check accepted state
What is the current approved version?
US3
Track obligations
What do we owe, to whom, by when?
US4
Generate report
Produce a site audit report from field evidence
US5
Capture field signal
Photo or voice from site — auto-structured
US6
Review and approve
Approve obligation closure or state changes
US7
Monitor deadlines
Alerts before obligations go overdue
US8
Connect channels
Link email, drive, messaging for auto-ingestion
Non-negotiable rules
State engine is always deterministic rules — never AI
Deviation detection is always a graph query — never AI
Every external or contractual action requires human approval before it executes
Every derived output carries provenance — no silent AI decisions, ever
Development Architecture Docs ↗
System layers
Presentation — Next.js 15 SSR, React, Tailwind CSS, shadcn/ui
API — FastAPI (REST + WebSocket), OAuth 2.0, JWT auth, OpenAPI docs
Orchestration — LangGraph state machine, PostgreSQL checkpoints, interrupt() HITL gates
Data — PostgreSQL 16 (relational) + Neo4j (graph + provenance) + Qdrant (vector search)
Workers — ARQ async workers over Redis. Parse → chunk → embed → index pipeline
Storage — S3/MinIO for raw files, photos, audio, exports
Ingress — Traefik (TLS 1.3, rate limiting). Dev: Docker Compose. Prod: Kubernetes

DesignUI · Screens · User types

Every screen must make provenance and source visible. Users should always understand why they see something and where it came from. No raw file dumps. No unexplained results.

Phase 0 — design now
Trusted Results UI — Ranked results showing: what it is, why it was returned, where it came from, its status.
Ask Attimo input — Query entry. Feel of asking a colleague, not running a search query.
Provenance drawer — Expands a result to show the full source chain: document → revision → approval → person.
Phase 1–2 — upcoming screens
HITL approval screens — 7 checkpoint types. Draft-first pattern for high-impact items.
Field signal capture — Ultra-light mobile view. Photo, voice, text. Minimal friction.
Obligation review UI — Confirm, edit, or reject extracted obligations from correspondence.
Document state timeline — Revision chain: accepted, pending, superseded.

MarketingAEC · Turkey + European markets
The problem
Construction projects generate thousands of signals that are never captured or linked to decisions. When disputes or audits happen, nobody has a single source of truth.
The solution
A coordination layer on top of existing tools. Connects fragmented signals into governed, traceable records — without forcing migration.
What it is not
Not a task managerNot a chatbotNot a document managerNot a PM toolNot a migration project
Target markets
Turkey AEC — Primary
Architecture, Engineering, Construction. High project volume, fragmented field coordination, strong need for dispute evidence.
EU AEC — Secondary
European firms with regulatory compliance, documentation standards, and handover evidence obligations.
5 value propositions
Find any project context in seconds — with proof of why it was returned
Always know the current approved version of any document or decision
Capture field signals automatically — photo, voice, text — without office software
Track every obligation with its source clause attached through to closure
Export audit-ready evidence packages for disputes, handovers, and claims instantly
